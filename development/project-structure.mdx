## Directory descriptions

### `src/components/`

Contains all React components split into two groups:

- **Section components** (`HeroSection.tsx`, `AboutSection.tsx`, etc.) — the seven full-page sections that make up the portfolio. Each is a self-contained component composed inside `pages/Index.tsx`.
- **`ui/`** — shadcn/ui primitives generated by the shadcn CLI (accordion, button, dialog, input, label, toast, tooltip, and more). Do not edit these files manually; re-run the CLI to update or add components.
- **`Navbar.tsx`** and **`Footer.tsx`** — the persistent layout components rendered around the sections.

### `src/hooks/`

Custom React hooks shared across components:

- `useScrollReveal.ts` — `IntersectionObserver`-based hook that reveals elements as they scroll into view.
- `use-mobile.tsx` — returns a boolean indicating whether the viewport is below the 768 px breakpoint.
- `use-toast.ts` — state management for the toast notification queue (generated by shadcn/ui).

### `src/lib/`

Utility functions. Currently contains a single export:

```typescript src/lib/utils.ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

`cn` merges Tailwind class lists, resolving conflicts correctly. Import it in any component that conditionally applies classes.

### `src/pages/`

Route-level components:

- `Index.tsx` — the single-page portfolio. Renders `Navbar`, all seven section components in order, and `Footer`.
- `NotFound.tsx` — the 404 fallback rendered for any unmatched route.

### `src/App.tsx`

The application root. Sets up global providers and the client-side router:

```tsx src/App.tsx
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { BrowserRouter, Route, Routes } from "react-router-dom";
import { Toaster as Sonner } from "@/components/ui/sonner";
import { Toaster } from "@/components/ui/toaster";
import { TooltipProvider } from "@/components/ui/tooltip";
import Index from "./pages/Index.tsx";
import NotFound from "./pages/NotFound.tsx";

const queryClient = new QueryClient();

const App = () => (
  <QueryClientProvider client={queryClient}>
    <TooltipProvider>
      <Toaster />
      <Sonner />
      <BrowserRouter>
        <Routes>
          <Route path="/" element={<Index />} />
          {/* ADD ALL CUSTOM ROUTES ABOVE THE CATCH-ALL "*" ROUTE */}
          <Route path="*" element={<NotFound />} />
        </Routes>
      </BrowserRouter>
    </TooltipProvider>
  </QueryClientProvider>
);

export default App;
```

Provider order matters:
1. `QueryClientProvider` — makes the TanStack Query client available to the whole tree.
2. `TooltipProvider` — required by Radix UI tooltip primitives.
3. `Toaster` / `Sonner` — renders the toast notification portals.
4. `BrowserRouter` — enables client-side routing.

### `src/main.tsx`

The Vite entry point. Renders `<App />` into the `#root` DOM element.

### `src/index.css`

Global stylesheet. Contains Tailwind base/components/utilities directives, CSS custom properties for design tokens (colors, radius, etc.), and any hand-written utility classes.

## Configuration files

| File | Purpose |
|---|---|
| `vite.config.ts` | Vite build config — dev server port (8080), `@` path alias, SWC React plugin, `lovable-tagger` in development mode. |
| `tailwind.config.ts` | Tailwind CSS configuration — content paths, theme extensions, and the `tailwindcss-animate` plugin. |
| `components.json` | shadcn/ui CLI configuration — component style, TypeScript, Tailwind path, and alias mappings used when running `npx shadcn add`. |
| `tsconfig.json` | TypeScript compiler options. |
| `package.json` | Dependencies and npm scripts. |
